Rendimiento del combustible en autos comerciales.

Realizaremos un anlisis de diferentes vehculos para poder realizar la prediccion sobre si la cantidad de cilindros de un auto es mayor a 4 cilindros(1) o menor o igual a 4 cilindros(0) a travs de una regresin logistica multiple.

Para realizar dicho estudio obtuvimos informacin del UCI Machine Learning Repository especficamente de Auto MPG de un estudio de Quinlan, R. (1993).

Dicha informacin cuenta con las siguientes variables:

MPG: Miles per galon, millas por galn, nuestra variable de inters.
Cylinders: La cantidad de cilindros del auto.
Displacement: Volumen total de aire y combustible que un auto puede desplazar en un ciclo de combustin.
Horsepower: Caballos de fuerza, es una medida de potencia.
Weight: Peso en libras.
Acceleration: Aceleracin.
Model year: Ao del modelo de auto.
Origin: Pas de origen del auto. 1: America, 2: Europa, 3: Asia
Car Name: Modelo del auto.

La base de datos tena algunos valores de la variable de inters como NaN, se eliminaron esas filas ya que considero que cualquier manera de conseguir ese valor aadira incertidumbre a nuestro modelo. Estaramos construyendo nuestro modelo en terreno que no es firme.

Ajustaremos los datos para poder realizar nuestro objetivo.

Ahora entrenaremos un modelo similar pero utilizando todos los datos de prueba, con ese modelo generaremos el vector de posibilidades para los datos de prueba, la matriz de confusin y reportaremos la exactitud, sensibilidad y especificidad del modelo con 3 umbrales: 0.5, 0.3 y 0.8.

In[5]:
from sklearn.metrics import confusion_matrix
from sklearn.metrics import ConfusionMatrixDisplay as cmd
from sklearn.metrics import accuracy_score
from sklearn.metrics import recall_score
import matplotlib.pyplot as plt

%matplotlib inline

# Entrenaremos el modelo de regresin logistica multiple
log_reg.fit(X_train, y_train)

# Obtenemos la probabilidad de y
y_probs = log_reg.predict_proba(X_test)[:,1]


# Definir tres umbrales para la clasificacin
umbrales = [0.3, 0.5, 0.8]

# Aqu guardaremos las matrices de confusin
cms = []

# Obtenemos datos por cada umbral
for umbral in umbrales:
    y_pred = (y_probs >= umbral).astype(int)
    cm = confusion_matrix(y_test, y_pred)
    accuracy = accuracy_score(y_test, y_pred)
    sensitivity = recall_score(y_test, y_pred)  # Sensibilidad (recall para clase 1)
    specificity = cm[0, 0] / (cm[0, 0] + cm[0, 1])  # Especificidad (TN / (TN + FP))
    cms.append(cm)
    print("Umbral: ", umbral, "\tAccuracy: ", accuracy, "\tSensitivity: ", sensitivity, "\tSpecificity: ", specificity)


#Imprimimos las matrices de confusin
disp = cmd(confusion_matrix=cms[0], display_labels=["4 o menos cilindros", "Ms de 4 cilindros"])
fig, ax = plt.subplots()
disp.plot(ax=ax, xticks_rotation="vertical")
ax.set_title("Matriz de Confusin para Umbral 0.3") 
plt.show()

disp = cmd(confusion_matrix=cms[1], display_labels=["4 o menos cilindros", "Ms de 4 cilindros"])

from sklearn.metrics import roc_curve, roc_auc_score

# Calcular la curva ROC
fpr, tpr, _ = roc_curve(y_test, y_probs)
auc_value = roc_auc_score(y_test, y_probs)

# Graficar la curva ROC
plt.figure(figsize=(8, 6))
plt.plot(fpr, tpr, label=f'ROC curve (AUC = {auc_value:.4f})')
plt.plot([0, 1], [0, 1], linestyle="--", color="gray")  # Lnea diagonal
plt.xlabel("False Positive Rate (1 - Specificity)")
plt.ylabel("True Positive Rate (Sensitivity)")
plt.title("Curva ROC - Regresin Logstica")
plt.legend()
plt.grid()
plt.show()

# Mostrar el valor de AUC
print("Valor AUC: ", auc_value)
